# Render Deployment Guide

## Prerequisites
1. GitHub account
2. Render account (free tier available)
3. MongoDB Atlas account (free tier available)

## Deployment Steps

### 1. Push to GitHub
```bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/yourusername/mobz-recharge.git
git push -u origin main
```

### 2. Set up MongoDB Atlas
1. Go to [MongoDB Atlas](https://cloud.mongodb.com/)
2. Create a free cluster
3. Create a database user
4. Get connection string
5. Whitelist all IPs (0.0.0.0/0) for Render

### 3. Deploy on Render
1. Go to [Render Dashboard](https://dashboard.render.com/)
2. Click "New" â†’ "Blueprint"
3. Connect your GitHub repository
4. Select the `day13` folder
5. Render will automatically detect the `render.yaml` file
6. Set environment variables:
   - `MONGODB_URI`: Your MongoDB Atlas connection string
   - `JWT_SECRET`: Will be auto-generated
7. Deploy!

### 4. Seed Database (Optional)
After deployment, you can seed the database:
1. Go to your backend service shell in Render
2. Run: `npm run seed`

## Environment Variables Needed

### Backend (.env)
- `MONGODB_URI`: MongoDB connection string
- `JWT_SECRET`: Auto-generated by Render
- `PORT`: 10000 (set by Render)
- `NODE_ENV`: production

### Frontend (.env.production)
- `VITE_API_URL`: https://your-backend-service.onrender.com

## URLs After Deployment
- Frontend: https://mobz-frontend.onrender.com
- Backend API: https://mobz-backend.onrender.com
- Health Check: https://mobz-backend.onrender.com/api/health

## Troubleshooting
1. Check logs in Render dashboard
2. Ensure MongoDB Atlas allows connections from 0.0.0.0/0
3. Verify environment variables are set correctly
4. Check that both services are running
